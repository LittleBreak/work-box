# wondermall-mos 技术架构升级决策报告

> 面向决策层 | 2026-03-04

---

## 1. 背景与动机

当前项目技术栈（Vue 2.7 + Webpack 3 + Babel 6）已接近生命周期末期：

- **Vue 2 已于 2023 年 12 月 31 日停止维护**，不再接收安全补丁
- Webpack 3、Babel 6 生态库逐步停更，新功能和性能优化无法享受
- Element UI 2 不支持 Vue 3，长期锁死技术演进
- 新人招聘和团队技术成长受限于过时技术栈

## 2. 项目规模

| 统计项          | 数量   |
| --------------- | ------ |
| Vue 文件总数    | 996 个 |
| Vuex Store 模块 | 124 个 |
| API 模块        | 77 个  |

## 3. 升级路径（两个方案共同部分）

无论选择哪个方案，以下两步是必须且相同的前置工作：

| 阶段       | 内容                          | 工作量   |
| ---------- | ----------------------------- | -------- |
| **Step 1** | Webpack 3 → 5 + Babel 6 → 7   | 5-8 人天 |
| **Step 2** | Vue Router 3 → 4 + Vuex 3 → 4 | 3-5 人天 |

两个方案从 Step 3 开始分叉。

## 4. 方案对比

### 方案一：全链路迁移（一步到位）

在共同步骤之后继续：

| 阶段     | 内容                                               | 工作量         |
| -------- | -------------------------------------------------- | -------------- |
| Step 3   | 切换 @vue/compat，逐文件消除 996 个 Vue 2 废弃写法 | 20-30 人天     |
| Step 4   | 移除 @vue/compat，切换纯 Vue 3                     | 2-3 人天       |
| Step 5   | Element UI 2 → Vue 3 UI 库（全量替换）             | 25-35 人天     |
| 其他     | 第三方库 + 监控 SDK 适配                           | 5-8 人天       |
| **合计** |                                                    | **60-89 人天** |

### 方案二：新旧共存（渐进式）

在共同步骤之后：

| 阶段     | 内容                                                            | 工作量         |
| -------- | --------------------------------------------------------------- | -------------- |
| Step 3   | 切换 @vue/compat + 双 UI 库配置（仅改入口和构建，不改业务代码） | 3-4 人天       |
| 其他     | vue-i18n + 监控 SDK 适配                                        | 2-3 人天       |
| **合计** |                                                                 | **13-20 人天** |

### 核心差异一览

| 维度           | 方案一                        | 方案二                       |
| -------------- | ----------------------------- | ---------------------------- |
| **总投入**     | 60-89 人天                    | 13-20 人天                   |
| **老页面处理** | 全量改造 996 个文件           | 不动，按需迁移               |
| **新页面能力** | Vue 3 + 新 UI 库              | Vue 3 + 新 UI 库（相同）     |
| **打包体积**   | 最优（纯 Vue 3 tree-shaking） | 较大（compat 层 + 双 UI 库） |
| **运行时性能** | 最优                          | 略有开销（compat 模拟层）    |
| **技术债**     | 一次性清零                    | 保留，逐步消化               |
| **回归风险**   | 高（改动面大）                | 低（老页面不动）             |
| **业务影响**   | 需要较长时间专项投入          | 可与业务迭代并行             |

## 5. 风险评估

### 共同风险

| 风险                                  | 应对                           |
| ------------------------------------- | ------------------------------ |
| Webpack 5 与现有 loader/plugin 不兼容 | Step 1 预留充足调试时间        |
| SGM/Sentry 监控 SDK 不兼容 Vue 3      | 提前联系京东 SGM 团队确认      |
| 迁移期间与业务开发分支冲突            | 约定迁移分支策略，缩短并行窗口 |

### 方案一特有风险

- Element UI 在 @vue/compat 下部分组件异常 → 需逐个排查 polyfill
- 自定义主题 CSS（88 个文件）在新 UI 库下大面积失效 → 需专人负责样式迁移
- 全量改造期间可能出现连锁回归 → 需充分的回归测试周期

### 方案二特有风险

- @vue/compat 官方可能停止维护 → 需关注 Vue 官方动态
- 双 UI 库并存导致首屏加载变大 → 通过路由懒加载缓解
- 新老页面 CSS 全局污染 → 新 UI 库使用 CSS Modules 隔离

## 6. 建议

| 场景                           | 推荐方案       | 理由                                                         |
| ------------------------------ | -------------- | ------------------------------------------------------------ |
| 业务需求密集，无法抽出大块人力 | **方案二**     | 13-20 天完成基础设施升级，新页面立即享受 Vue 3，老页面零风险 |
| 有专项技术升级排期（2-3 个月） | **方案一**     | 一次性解决技术债，长期收益最大                               |
| 不确定 / 想先试水              | **方案二先行** | 方案二是方案一的子集，后续可随时平滑过渡到方案一终态         |

**关键结论**：方案二不是妥协，而是务实的分步策略。完成方案二后，项目已运行在 Vue 3 运行时上，新页面使用 Vue 3 全部能力，后续按业务节奏逐模块迁移老页面，最终可达到与方案一相同的终态。
